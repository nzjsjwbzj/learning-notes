```
看到 CON 结尾，就是 Control (控制寄存器)。
看到 MOD 结尾，就是 Mode (模式寄存器)。
看到 E 开头 (在IE寄存器里)，就是 Enable (允许)。
看到 P 开头 (在IP寄存器里)，就是 Priority (优先级)。
看到 H 或 L 结尾，就是 High (高8位) 或 Low (低8位)。
看到 S 开头，通常指 Serial (串口)。
看到 T 开头，通常指 Timer (定时器)。
```

# 位寻址

有一个简单的规律（适用于标准 51）：

只有地址能被 8 整除的寄存器，才可以位寻址。 (即地址的最后一位是 0 或 8 ，如 0x80 , 0x88 , 0x90 , 0xA8 等

“堆数模电，无法单恋”

- 堆 : SP (堆栈)
- 数 : SBUF , DPTR , TH/TL (所有跟数据、数值有关的)
- 模 : TMOD (模式)
- 电 : PCON (电源)
- 无法单恋 : 不能 单独操作某一位（位寻址）





在 51 单片机的特殊功能寄存器 (SFR) 中， 不能进行位寻址 的主要有：

1. TMOD (定时器模式寄存器) —— 最常考/最容易错的一个！
2. SBUF (串口数据缓冲寄存器)
3. TH0 / TL0 / TH1 / TL1 (定时器计数值寄存器)
4. SP (堆栈指针)
5. DPTR (DPH / DPL) (数据指针)
6. PCON (电源控制寄存器)
### 怎么快速判断？
有一个简单的规律（适用于标准 51）：

只有地址能被 8 整除的寄存器，才可以位寻址。 (即地址的最后一位是 0 或 8 ，如 0x80 , 0x88 , 0x90 , 0xA8 等)



### 为什么 TMOD 不能位寻址特别重要？
在写代码时，这是一个非常典型的坑：

- TCON (可位寻址): 你可以写 TR1 = 1; 单独启动定时器1，不影响定时器0。
- TMOD (不可位寻址): 你 不能 写 TMOD.1 = 1; 或者类似的操作。
  - 如果你只想修改定时器1的模式，而不改变定时器0的设置，你必须用位运算：
  - TMOD = (TMOD & 0x0F) | 0x20; (保留低4位，修改高4位)
  - 但在简单程序中，通常直接 TMOD = 0x20; 一次性赋值，这会覆盖掉定时器0的设置。
  总结： 最需要记住的“不可位寻址”寄存器就是 TMOD ，因为它里面的配置位很多且混在一起，却只能整体操作。



为什么不能位寻址

这是一个非常好的底层问题！为什么有的寄存器能一位一位地操作（位寻址），有的却只能整个字节一起操作？

根本原因在于 51 单片机的 CPU 指令集架构 (Instruction Set Architecture) 设计 和 硬件电路成本 的权衡。

### 1. 指令集的限制 (硬件听不懂)
位寻址不是“想有就能有”的，它需要 CPU 有专门的 指令 来支持。

- 可位寻址区: CPU 有专门的指令，比如 SETB (置位)、 CLR (清零)、 CPL (取反)、 JB (位跳转)。这些指令的操作数可以直接是一个“位地址”。
  - 例如： SETB 0x8E (把地址 0x88 寄存器的第 6 位 TR1 置 1)。
- 不可位寻址区: CPU 没有设计 针对这些地址的位操作指令。如果你强行写 TMOD.5 = 1 ，编译器翻译不成对应的机器码，因为它找不到一条指令叫“去把 0x89 地址的第 5 位改成 1”。
### 2. 内存映射空间不够 (地址不够用)
51 单片机的位地址空间是有限且经过精心规划的。

- 位地址空间 (Bit Address Space): 只有 128 个 位于 RAM (0x20-0x2F) 的位，以及 128 个 位于 SFR (0x80-0xFF) 的位。
- SFR 区域的规则: 为了简化硬件译码电路的设计，Intel 的工程师规定：
  - SFR 地址能被 8 整除 ( 0x80, 0x88, 0x90... ) 的这 16 个字节，它们的每一位都被映射到了位地址空间。
  - 地址 不能 被 8 整除的（比如 TMOD=0x89 ），它们的位就没有被分配“位地址”。





### 片内 RAM 详细地图
我们可以把片内 RAM 分成三块区域：

1. 工作寄存器组区 (0x00 - 0x1F)
   
   - 32 字节。
   - 分为 4 组（Bank 0-3），每组 8 个寄存器 (R0-R7)。
   - 不可直接位寻址 (虽然可以通过字节地址访问，但没有分配专门的位地址)。
2. 👉 位寻址区 (0x20 - 0x2F)
   
   - 重点在这里！
   - 这 16 个字节 (16 bytes × 8 bits = 128 bits) 是特权区域。
   - 你既可以按 字节 访问它们（例如 mov A, 0x20 ），也可以按 位 访问它们（例如 SETB 00H ）。
   - 这里的“位地址”范围是 0x00 到 0x7F 。
   - C语言中的 bdata 关键字 就是把变量存在这里。
3. 通用 RAM 区 (0x30 - 0x7F)
   
   - 80 字节。
   - 这就是普通的内存，存放你的全局变量、数组、堆栈等。
   - 不可位寻址 。

# IO口





### 1. P0 口 (Port 0) —— 最特殊的一个
- 结构特点： 开漏输出 (Open-Drain) 。
  - 它的内部输出级主要是一个 N-MOS 管。
  - 当输出低电平 (0) 时，MOS 管导通，直接接地，拉电流能力很强（可以驱动 LED）。
  - 当输出高电平 (1) 时，MOS 管截止。由于 没有内部上拉电阻 ，引脚处于“悬空” (高阻抗) 状态。它 不能 输出高电平去驱动外部电路。
- 用途：
  1. 作为普通 I/O 口： 必须外接上拉电阻 （通常 4.7kΩ 或 10kΩ）才能输出高电平。
  2. 作为地址/数据总线 (AD0-AD7)： 当你外扩存储器时，P0 口分时复用输出低 8 位地址和 8 位数据。在这种模式下，内部会有强推挽驱动，不需要外接上拉电阻。
- 一句话记忆： P0 是“没力气”的，必须外挂电阻才能拉高。
### 2. P1 口 (Port 1) —— 最普通的一个
- 结构特点： 准双向口 (Quasi-Bidirectional) 。
  - 内部带有一个 弱上拉电阻 （约 20kΩ - 40kΩ）。
  - 输出高电平 (1) 时，通过这个电阻拉高，驱动能力很弱（只能驱动几个微安的负载）。
  - 输出低电平 (0) 时，内部 MOS 管接地，驱动能力较强。
- 用途： 纯粹的通用 I/O 口，通常用于接 LED（灌电流驱动）、按钮等。
- 一句话记忆： P1 是标准的，自带电阻，够用。
### 3. P2 口 (Port 2) —— 高位地址
- 结构特点： 准双向口 （同 P1，带内部上拉）。
- 用途：
  1. 作为普通 I/O 口： 和 P1 完全一样。
  2. 作为地址总线 (A8-A15)： 当外扩存储器时，它输出高 8 位地址。
- 一句话记忆： P2 也是带电阻的，偶尔客串高位地址。
### 4. P3 口 (Port 3) —— 功能最强
- 结构特点： 准双向口 （同 P1，带内部上拉）。
- 用途：
  1. 作为普通 I/O 口： 和 P1 一样。
  2. 第二功能 (复用功能)： 这是 P3 最重要的地方！它的每一位都有特殊职责：
     - P3.0 : RXD (串口接收)
     - P3.1 : TXD (串口发送)
     - P3.2 : INT0 (外部中断0)
     - P3.3 : INT1 (外部中断1)
     - P3.4 : T0 (定时器0输入)
     - P3.5 : T1 (定时器1输入)
     - P3.6 : WR (外部写信号)
     - P3.7 : RD (外部读信号)
- 一句话记忆： P3 带电阻，身兼数职，全是重要外设接口。







```c
#include <reg51.h>

void main(){
    // 初始状态：假设P1口当前值为0xFF
    
    // 方式B：字节操作
    sfr PORT1 = 0x90;
    PORT1 = 0xFE;    // 结果：P1 = 1111 1110
    
    // 方式C：位操作
    sbit led = 0x90; // P1.0
    led = 0;         // 结果：P1 = 1111 1110（仅第0位变0）
    
    // 方式D：位操作（推荐）
    sbit led2 = P1^1; // P1.1
    led2 = 0;         // 结果：P1 = 1111 1100（第0位和第1位为0）
}
```







## 进阶-什么时候作为地址数据线

不用手动配置

**51单片机通过硬件自动检测指令类型来决定P0和P2的功能**：

1. **普通I/O指令**（MOV P0, #data等）：P0/P2作为通用I/O口
2. **MOVX指令**（访问外部RAM）：P0作为地址/数据总线，P2作为高8位地址
3. **MOVC指令且EA=0**（访问外部ROM）：同上

```c
// 1. 普通I/O操作（P0/P2作为I/O口）
P0 = 0x55;                    // 对应 MOV P0, #55H
unsigned char data = P2;      // 对应 MOV A, P2

// 2. 访问外部RAM（P0/P2作为总线）
unsigned char xdata *ptr = 0x1234;
*ptr = 0xAA;                  // 对应 MOVX @DPTR,A

// 3. 访问外部ROM（P0/P2作为总线）
unsigned char code *cptr = 0x1000;
data = *cptr;                 // 对应 MOVC A,@A+DPTR
```

## 方式一 

16位手动

```c
#include <reg51.h>

sbit WAVE = P1^0;     // 方波输出引脚
unsigned char count = 0;  // 中断次数计数器

void main(){
    // 定时器0初始化 - 方式1，16位定时
    TMOD = 0x01;        // 定时器0方式1
    
    // 计算25ms定时初值（24MHz晶振）
    // 需要计数值 = 25ms / 0.5μs = 50000
    // 初值 = 65536 - 50000 = 15536
    TH0 = (65536-50000) / 256;  // 高8位
    TL0 = (65536-50000) % 256;  // 低8位
    
    // 中断设置
    ET0 = 1;            // 允许定时器0中断
    EA = 1;             // 开总中断
    TR0 = 1;            // 启动定时器0
    
    WAVE = 0;           // 初始电平（假设低电平开始）
    count = 0;
    
    while(1);           // 主循环等待中断
}

// 定时器0中断服务函数
void timer0_isr() interrupt 1 {
    // 重装初值（25ms）
    TH0 = (65536-50000) / 256;
    TL0 = (65536-50000) % 256;
    
    count++;            // 中断次数加1
    
    if(count == 2){     // 2 × 25ms = 50ms
        count = 0;      // 计数器清零
        WAVE = ~WAVE;   // 翻转电平，产生方波
    }
}
```



## 方式二



八位自动重装



```c
#include <reg51.h>

sbit LED1 = P1^0;
sbit LED2 = P1^1;
unsigned char ms_count = 0;

void main(){
    // 方式2：8位自动重装
    TMOD = 0x02;        // T0方式2
    
    // 计算：定时250μs @12MHz
    // 250μs需要250个机器周期
    // 初值 = 256 - 250 = 6 = 0x06
    TH0 = 0x06;         // 重装值
    TL0 = 0x06;         // 初始计数值
    
    ET0 = 1;
    EA = 1;
    TR0 = 1;
    
    // LED1每1ms闪烁（快）
    // LED2每500ms闪烁（慢）
    while(1);
}

void timer0_isr() interrupt 1 {
    // 注意：方式2不需要手动重装TH0/TL0！
    // 硬件会自动将TH0的值装入TL0
    
    ms_count++;         // 250μs计数
    
    // 每4次中断=1ms
    if(ms_count % 4 == 0){
        LED1 = ~LED1;   // 1ms翻转一次
    }
    
    // 每2000次中断=500ms
    if(ms_count == 2000){
        ms_count = 0;
        LED2 = ~LED2;   // 500ms翻转一次
    }
}
```

```c
// 包含51单片机寄存器定义头文件
// 这个文件定义了P0、P1、TCON、TMOD等所有特殊功能寄存器
#include <reg51.h>

// T0方式1：用于精确长时间定时（控制LED闪烁）
// T1方式2：用于串口波特率发生器（提供串口通信的时钟）

// 定义LED连接到P1口的第0位
// sbit是51单片机特有的关键字，用于定义可位寻址的位
sbit LED = P1^0;

// 串口初始化函数
// 功能：配置串口通信参数，设置定时器1为波特率发生器
void uart_init(){
    // 设置定时器1为工作方式2（8位自动重装）
    // TMOD |= 0x20：使用或运算，只修改高4位（定时器1），不影响低4位（定时器0）
    // 0x20 = 0010 0000，高4位中的M1=1,M0=0表示方式2
    TMOD |= 0x20;
    
    // 配置波特率为9600 @11.0592MHz晶振
    // TH1和TL1都设置为0xFD，这是计算出的波特率初值
    // 计算过程：256 - 11059200/(9600×32×12) = 253 = 0xFD
    TH1 = 0xFD;    // 定时器1高字节（重装值）
    TL1 = 0xFD;    // 定时器1低字节（初始计数值）
    
    // 配置串口控制寄存器SCON
    // 0x50 = 0101 0000
    // SM0=0, SM1=1：串口工作方式1（8位UART，波特率可变）
    // REN=1：允许串口接收
    SCON = 0x50;
    
    // 电源控制寄存器PCON
    // 0x00 = 0000 0000
    // SMOD=0：波特率不倍增（如果SMOD=1则波特率加倍）
    PCON = 0x00;
    
    // TR1 = 1：启动定时器1计数
    // 定时器1开始工作，为串口提供波特率时钟
    TR1 = 1;
    
    // ES = 1：允许串口中断
    // 当串口发送完成或接收到数据时，会产生中断
    ES = 1;
}

// 定时器0初始化函数
// 功能：配置定时器0为50ms定时器，用于控制LED闪烁
void timer0_init(){
    // 设置定时器0为工作方式1（16位定时器）
    // TMOD |= 0x01：使用或运算，只修改低4位（定时器0）
    // 0x01 = 0000 0001，低4位中的M1=0,M0=1表示方式1
    TMOD |= 0x01;
    
    // 计算50ms定时初值（基于12MHz晶振）
    // 12MHz晶振，机器周期 = 12/12MHz = 1μs
    // 50ms = 50000μs，需要50000个机器周期
    // 16位定时器最大65536，初值 = 65536 - 50000 = 15536
    // 15536 = 0x3CB0，所以：
    TH0 = (65536-50000)/256;  // 0x3C（高8位）
    TL0 = (65536-50000)%256;  // 0xB0（低8位）
    
    // ET0 = 1：允许定时器0中断
    // 当定时器0溢出时（计满65536），会产生中断
    ET0 = 1;
    
    // TR0 = 1：启动定时器0计数
    // 定时器0开始从初值15536向上计数
    TR0 = 1;
}

// 主函数，程序入口点
void main(){
    // 调用初始化函数，配置硬件
    uart_init();    // 初始化串口
    timer0_init();  // 初始化定时器0
    
    // EA = 1：开启总中断（全局中断允许）
    // 这是中断系统的总开关，必须打开才能响应任何中断
    EA = 1;
    
    // 发送启动消息到串口
    // SBUF是串口数据缓冲寄存器，写入数据就会启动发送
    SBUF = 'S';      // 发送字符'S'，表示系统启动
    
    // while(!TI);：等待发送完成
    // TI是发送中断标志，发送开始时TI=0，发送完成后硬件置TI=1
    // 这个循环会一直等待，直到TI=1才继续执行
    while(!TI);
    
    // TI = 0：清除发送中断标志
    // 必须手动清除TI，否则会错误地认为一直在发送
    TI = 0;
    
    // 主循环，程序主要在这里运行
    // while(1)是无限循环，单片机程序通常都是这样
    // 实际工作由中断服务函数完成，主循环可以处理其他任务
    while(1){
        // 这里可以添加其他需要持续执行的代码
        // 比如：按键检测、数据处理等
        // 当前为空，所有工作由中断处理
    };
}

// 定时器0中断服务函数
// interrupt 1 表示这是中断号1的中断服务函数
// 51单片机中断号：0-外中断0, 1-定时器0, 2-外中断1, 3-定时器1, 4-串口
void timer0_isr() interrupt 1 {
    // 静态局部变量，只初始化一次，函数退出后值保留
    // 用于计数中断次数，实现更长的定时
    static unsigned char count = 0;
    
    // 重装定时器0初值（方式1需要手动重装）
    // 如果不重装，下次定时时间会错误
    TH0 = (65536-50000)/256;
    TL0 = (65536-50000)%256;
    
    // 中断次数加1
    count++;
    
    // 判断是否达到2次中断（2×50ms = 100ms）
    if(count == 2){
        count = 0;      // 计数器清零，重新开始计数
        LED = ~LED;     // LED状态取反，实现闪烁
        // ~是按位取反运算符，0变1，1变0
        // 假设LED低电平点亮，那么这会交替点亮和熄灭LED
    }
}

// 串口中断服务函数
// interrupt 4 表示这是中断号4的中断服务函数（串口中断）
void uart_isr() interrupt 4 {
    // 判断是否是接收中断（RI == 1）
    // RI是接收中断标志，收到一个字节后硬件置1
    if(RI == 1){
        RI = 0;    // 必须手动清除接收中断标志
        
        // 读取接收到的数据
        // SBUF是双缓冲寄存器，读取时是接收缓冲区
        unsigned char data = SBUF;
        
        // 将接收到的数据原样发送回去（回显功能）
        // 写入SBUF会启动发送过程
        SBUF = data;
    }
    
    // 判断是否是发送中断（TI == 1）
    // TI是发送中断标志，发送完成时硬件置1
    if(TI == 1){
        TI = 0;    // 必须手动清除发送中断标志
        // 这里可以添加发送完成后的处理代码
        // 比如：准备下一字节发送、更新状态等
    }
}
```

## 通信-第九位

![image-20251219122237078](./51%E5%BA%94%E8%AF%95.assets/image-20251219122237078.png)

> **之所以叫“第 9 位”，不是因为 SBUF 里真的有 9 位数据，
>  而是因为在“串行比特流的发送顺序中，它排在 8 位数据之后”。**

下面我按**从“物理比特流 → 硬件寄存器 → 命名习惯”**三层，把它彻底说清楚。

------

### 一、从“线上真正发了什么”看：它确实是第 9 个数据位

以 **方式 2 / 方式 3** 为例，发送顺序是：

```
起始位 → D0 → D1 → D2 → D3 → D4 → D5 → D6 → D7 → D8 → 停止位
```

- `D0 ~ D7`：来自 `SBUF`
- `D8`：来自 `SCON.TB8`

👉 **在“数据位序列”里，它排第 9 个**
 所以在**通信语义上**，它就是“第 9 位数据”。



### 第 9 位到底“是什么用的”（一句话定性）

> **第 9 位不是为了扩展数据位数，而是为了“通信控制”**

### 常见用途（用户自定义）：

- 地址 / 数据区分
- 多机通信唤醒
- 主从标志
- 校验位（非常少见）

例如：

```c
TB8 = 1;   // 地址帧
SBUF = addr;

TB8 = 0;   // 数据帧
SBUF = data;
```





## 例子

### 关键硬件规则（非常重要）

> 当 `SM2 = 1` 时：
>  **只有第 9 位 = 1 的帧，才会置位 RI**

也就是说：

- 地址帧（第9位=1）→ 才能“进中断 / 轮询”
- 数据帧（第9位=0）→ **硬件直接丢弃**v

```c
void master_send(void)
{
    TB8 = 1;         // 地址帧
    SBUF = 0x01;     // 从机地址
    while (!TI); TI = 0;

    TB8 = 0;         // 数据帧
    SBUF = 0x55;     // 数据
    while (!TI); TI = 0;
}


#define MY_ADDR 0x01

void slave_init(void)
{
    SCON = 0xD0;
    /*
      SM0=1 SM1=0 → 方式2
      SM2=1        → 只接收第9位=1
      REN=1
    */
}



void slave_recv(void)
{
    if (RI)
    {
        RI = 0;

        if (RB8)   // 第9位=1 → 地址帧
        {
            if (SBUF == MY_ADDR)
            {
                SM2 = 0;    // 地址匹配，打开数据接收
            }
            else
            {
                SM2 = 1;    // 地址不匹配，继续过滤
            }
        }
        else        // 第9位=0 → 数据帧
        {
            unsigned char data = SBUF;
            // 处理数据
            SM2 = 1;        // 收完重新进入过滤
        }
    }
}

```



当 SM2=0 时，串口不进行多机通信判断，
 硬件规定只要接收到一帧有效数据，RB8 即被置 1，
 此时 RB8 不再反映第 9 位的实际值。





### TB8 的特点（记死）

- 只在 **方式 2 / 方式 3**
- **永远参与发送**
- **和 SM2 没任何关系**

> 👉 不管 SM2 是 0 还是 1，
>  **TB8 都会作为第 9 位被发出去**

------

## 三、RB8 —— “收到的第 9 位”，但要看 SM2

RB8 **本来是**：

> “接收到的第 9 位”

但 **SM2 会改变 RB8 的意义**。

------

## 情况一：SM2 = 1（多机通信模式，重点）

### 硬件规则（这是核心）

> 当 SM2 = 1：
>  **只有第 9 位 = 1 的帧，才能被“接收”**

也就是说：

| 接收到的第9位 | 结果                    |
| ------------- | ----------------------- |
| 1             | RI = 1，RB8 = 1         |
| 0             | **直接丢弃，RI 不置位** |

👉 你 **根本收不到** 第 9 位 = 0 的帧

### 结论

- RB8 = 1 → 一定是地址帧
- RB8 = 0 → **不存在这种情况**

------

## 情况二：SM2 = 0（普通接收模式，本题易错点）

当 SM2 = 0：

- 串口不做多机判断
- **所有帧都接收**

但此时：

> **RB8 不再表示真实的第 9 位**

硬件规定：

| 结果         | RB8  |
| ------------ | ---- |
| 成功接收一帧 | 1    |

👉 RB8 **退化成“接收成功标志”**

RI也会置位



## 总结

SM是0时，接收到后RI置位1，RB8置1

SM是1时，只有TB8是1才接受，然后RI=1，RB8=1；不然就抛弃

# 频率

![image-20251218184036814](./51%E5%BA%94%E8%AF%95.assets/image-20251218184036814.png)

51单片机规定：**外部脉冲的最高频率 ≤ 振荡频率 / 24**

为什么？

- 机器周期 = 12个时钟周期
- 每个机器周期采样一次外部引脚
- 为了可靠检测，需要采样至少2次才能确认一个脉冲
- 所以：最高频率 = 振荡频率 / (12 × 2) = 振荡频率 / 24



## UART帧结构

51单片机UART的4种工作方式：

1. **方式0**：同步移位寄存器（无起始/停止位）
2. **方式1**：**8位UART，无校验位**（1起始+8数据+1停止）
3. **方式2**：9位UART，固定波特率（1起始+8数据+1校验/控制+1停止）
4. **方式3**：9位UART，可变波特率（1起始+8数据+1校验/控制+1停止）

**注意**：默认常用的是方式1，没有校验位。

# 单总线协议

## 读取

```c
// 假设 d 是用来存放读取数据的变量 (unsigned char d)
// 假设 DQ 是单总线的 GPIO 引脚位定义

for(i=8; i>0; i--) {    // 循环 8 次，因为 1 个字节 = 8 个位 (Bit)

    /* 
     * 1. 移位准备 (LSB First 策略)
     * 单总线是“低位先行”，即最先读到的是最低位(Bit0)。
     * 这里的逻辑是：先把 d 里的数据整体右移一位，空出最高位(Bit7)，
     * 等会儿读到的新数据就填在最高位。循环8次后，最初读到的位就会被移到最低位。
     */
    d = d >> 1;  

    /* 
     * 2. 启动读时隙 (Initiate Read Time Slot)
     * 所有通信都由主机发起。主机通过“拉低-释放”的动作告诉从机：
     * “请把这一位的数据放到总线上！”
     */
    DQ = 0;                     // 主机拉低总线
    delay = 1; while(--delay);  // 保持微秒级延时 (>1us)，作为一个起始脉冲
    DQ = 1;                     // 主机释放总线（拉高），准备采样
                                // 此时，如果从机想发“0”，它会把线拉低；想发“1”，它就保持线为高。

    /* 
     * 3. 等待并采样 (Sampling)
     * 单总线协议要求：主机必须在拉低总线后的 15us 内完成读取。
     */
    delay = 4; while(--delay);  // 等待一小会儿（通常约 10us），到达最佳采样点
    
    if(DQ) {                    // 主机读取 DQ 引脚电平
        d = d + 0x80;           // 如果读到 1 (高电平)，将最高位 (Bit7) 置为 1
                                // 如果读到 0，这一行不执行，最高位默认为 0
    }

    /* 
     * 4. 完成时隙 (End Time Slot)
     * 协议规定读一位数据的完整周期至少需要 60us。
     * 刚才读取只用了十几微秒，剩下的时间要通过延时补足。
     */
    delay = 23; while(--delay); // 延时约 45-50us，让时间隙完整结束

    DQ = 1;                     // 确保释放总线，准备进入下一轮循环
}

// 循环结束后，d 中就是读取到的完整字节
```



## 写入

```C
/*
 * 函数名：Write_Byte
 * 功能：向单总线设备写入一个字节
 * 参数：dat - 要写入的数据 (unsigned char)
 */
void Write_Byte(unsigned char dat)
{
    unsigned char i;
    
    for(i=0; i<8; i++)          // 循环 8 次，写 8 个位
    {
        /* 1. 启动写时隙 */
        DQ = 0;                 // 主机拉低总线，通知从机“我要写了”
        delay = 1; while(--delay); // 保持极短时间 (约 1-2us)
                                // 这个短暂的拉低是同步信号
        
        /* 2. 根据数据决定总线电平 */
        // 我们要写 dat 的最低位 (dat & 0x01)
        if(dat & 0x01)          
        {
            DQ = 1;             // 【写 1】：马上释放总线！
                                // 线被电阻拉高，从机采样到高电平
        }
        else
        {
            DQ = 0;             // 【写 0】：继续按住总线！
                                // 保持低电平，从机采样到低电平
        }

        /* 3. 保持写状态 (Write Time Slot) */
        delay = 10; while(--delay); // 【关键】延时约 60us
                                    // 从机通常在 15-60us 之间进行采样
                                    // 所以这个状态（高或低）必须维持住

        /* 4. 恢复与移位 */
        DQ = 1;                 // 释放总线，为下一位做准备
        dat = dat >> 1;         // 数据右移，把下一位挪到最低位准备发送
        
        delay = 1;              // 稍微延时，作为两位数据之间的恢复时间
    }
}
```



## 初始化

```c
/*
 * 函数名：Init_1Wire
 * 功能：复位单总线设备，并检测设备是否存在
 * 返回值：0 - 设备存在 (Success)
 *         1 - 设备不存在 (Error)
 */
bit Init_1Wire(void)
{
    bit presence; // 用于存储从机的应答结果

    /* 1. 主机发送复位脉冲 (Reset Pulse) */
    DQ = 0;                     // 主机拉低总线
    delay = 80; while(--delay); // 【关键】必须拉低至少 480us
                                // (假设这里的 80 循环能达到这个时间)
    
    DQ = 1;                     // 主机释放总线（拉高）
    
    /* 2. 主机等待从机响应 */
    delay = 10; while(--delay); // 等待约 15-60us
                                // 给从机一点时间反应，等它把线拉低

    /* 3. 采样存在脉冲 (Presence Pulse) */
    presence = DQ;              // 读取 DQ 状态
                                // 如果读到 0，说明有设备把线拉低了（设备存在）
                                // 如果读到 1，说明线上没人理你（设备不存在）

    /* 4. 等待时隙结束 */
    delay = 20; while(--delay); // 必须延时足够长，让整个复位周期完成
                                // (通常至少还要等 400us 左右)
    
    DQ = 1;                     // 确保最后总线是释放状态
    
    return presence;            // 返回检测结果
}
```

# IIC地址

I²C 7 位地址格式（以 PCF8591 为例）：
PCF8591 的固定高 4 位 = `1001`，低 3 位由引脚 **A2、A1、A0** 电平决定（接地为 0，接 +5V 为 1）。

完整 7 位地址 = `1001 A2 A1 A0`（二进制）。

然后，I²C 协议发送时，是 **7 位地址 + 1 位 R/W** 组成一个字节：

- 写：`(7 位地址 << 1) | 0`
- 读：`(7 位地址 << 1) | 1`
