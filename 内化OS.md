# 第七章 I/O



## **轮询、中断、DMA、通道I/O的区别**

- ### **通道方式是什么？**

  **通道方式**是计算机系统中一种高效的输入输出（I/O）处理方式。它通过一个**专门的硬件组件**——**通道**，来负责管理外部设备的输入输出操作，从而解放CPU，让CPU可以专注于其他任务。

  通道方式的核心思想是让一个“帮手”（通道）去完成所有的繁琐I/O工作，而不是让“老板”（CPU）亲自管理每一个细节。

  ------

  ### **通俗举例：理解通道方式**

  想象一个工厂的管理模式：

  - **CPU**是工厂的老板，负责整个工厂的运营管理。
  - **通道**是一个专业的主管（帮手），专门负责协调货物的进出和物流。
  - **I/O设备**是工厂的供货商或运输工具（如卡车、船）。

  在通道方式中，流程是这样的：

  1. **老板（CPU）下达任务**：告诉主管（通道）去协调供货商完成货物运输。
  2. **主管（通道）独立工作**：主管去安排供货商发货，处理货物到达，完成整个运输任务。
  3. **任务完成通知老板**：当货物到达时，主管向老板汇报“任务完成”，老板只需要简单确认。

  通过这样的分工，**老板不用亲自处理物流细节**，大大提高了工厂的效率。

  ------

  ### **通道方式与轮询、中断、DMA的区别**

  | **方式** | **核心思想**                                                 | **CPU参与程度** | **效率** | **适用场景**                              |
  | -------- | ------------------------------------------------------------ | --------------- | -------- | ----------------------------------------- |
  | **轮询** | CPU主动询问设备是否需要服务，反复检查状态                    | 高              | 最低     | 简单设备，低速、低实时性需求              |
  | **中断** | 设备准备好后主动向CPU发信号，通知CPU来处理请求               | 中              | 中等     | 实时响应场景，如键盘输入、鼠标点击等      |
  | **DMA**  | DMA控制器负责直接在内存和设备间传输数据，CPU只在开始和结束时参与 | 低              | 高       | 大量数据传输，如磁盘读写、音视频流处理等  |
  | **通道** | 通道独立管理设备，负责所有I/O操作，CPU只需要下达命令和接收结果 | 最低            | 最高     | 大型计算机系统，多个外设并行、复杂I/O场景 |

  ------

  ### **更详细的解释：逐一对比**

  #### **1. 轮询：CPU亲自处理一切**

  - **通俗描述**：轮询就像老板亲自守在门口，不停地问快递员“快递到了吗？”，直到快递员说“到了”。
  - **缺点**：老板（CPU）花费大量时间在等待，无法专注于更重要的事情。
  - **适用场景**：适合简单的设备（如键盘），设备响应速度快且对效率要求不高。也可以是中断

  ------

  #### **2. 中断：设备主动打断CPU**

  - **通俗描述**：中断就像老板在办公室忙自己的事情，快递员来了按门铃（发送中断信号），老板停下工作来接收快递。
  - **优点**：老板不用反复跑去门口检查，效率有所提高。
  - **缺点**：如果门铃响得太频繁，老板的工作会被频繁打断（中断过多会影响CPU性能）。
  - **适用场景**：适合实时响应的设备（如鼠标、网络数据包）。

  ------

  #### **3. DMA：找个小帮手做数据搬运**

  ![图片](https://img-blog.csdnimg.cn/img_convert/45fb891720ebd47f99dbddee41762b2b.png)

  - **通俗描述**：DMA就像老板找了个快递员帮手（DMA控制器），让他负责把快递从门口搬到仓库，自己只需要在开始和结束时确认。
  - **优点**：减少老板参与，提升数据搬运效率。
  - **缺点**：只能完成简单的搬运任务，帮手需要老板每次单独指派。
  - **适用场景**：适合大量数据的传输，如磁盘到内存。

  ------

  #### **4. 通道：一个专业的主管处理I/O**

  - **通俗描述**：通道方式就像老板雇了一个专业主管（通道），不但能自己安排搬运任务，还能管理多个快递员（I/O设备），把事情处理得井井有条，最后再向老板汇报结果。

  - 优点

    ：

    - 专业主管可以独立管理多个设备（并行操作），CPU几乎不参与。
    - 更适合复杂的任务，比如同时处理多个设备的I/O请求。

  - 缺点

    ：

    - 需要增加通道硬件，成本较高。

  - **适用场景**：适合高性能计算、大型系统（如服务器、主机），需要管理多个设备并处理复杂I/O任务。

  ------

  ### **总结：通道方式的优点**

  - **效率最高**：通道独立完成所有I/O任务，CPU负担最小。
  - **并行性强**：一个通道可以同时管理多个I/O设备，支持并行处理。
  - **复杂任务处理能力强**：适用于大规模数据处理和复杂的设备交互场景。

  ------

  ### **总结性类比：四种方式的“老板与物流”故事**

  1. **轮询**：老板自己天天跑门口检查快递，累得要死。
  2. **中断**：快递员来了按门铃，老板暂停工作去处理，稍微轻松些。
  3. **DMA**：雇了个搬运工帮忙搬快递，老板只用指挥开始和结束。
  4. **通道**：请了个主管，直接把物流全权交给他，老板只需要等最终结果。

  通过这个类比，可以清楚地看出通道方式是最省心、省力、高效的方式，但实现也需要更高的硬件成本和复杂性。

## 设备控制器、I/O通道、设备驱动、设备 之间是什么关系，他们之间是如何配合工作的？

设备： 这是实际的硬件，如打印机、硬盘驱动器、键盘等。它们是计算机硬件系统的重要组成部分。

设备控制器： 这是一个硬件组件，它的任务是管理和控制一个或多个设备的操作。例如，磁盘控制器负责管理磁盘驱动器的操作，包括数据的读写等。设备控制器通常与设备紧密集成在一起。

I/O通道： I/O通道也被称为I/O子系统，它是一种特殊的硬件结构，用于处理I/O操作，并减轻CPU的负担。它管理着CPU与设备控制器之间的通信。

设备驱动： 这是一个软件组件，它允许操作系统和其他计算机程序与特定硬件设备进行交互。设备驱动程序翻译高级操作系统指令为能被设备控制器理解的低级指令。

### **字节多路通道**

- **核心点**：字节多路通道是专门服务**低速设备**（比如键盘、打印机）的。
- 它会按顺序**一个个地检查设备**，看看有没有需要处理的数据。
- 一次只处理**一个设备的一小部分数据**（比如一个字节），然后迅速切换到下一个设备。

#### **为什么是多路的？**

因为通道在**多个设备之间切换**，用一个通道服务多个低速设备。

#### **特点**：

1. 适合处理**小数据量**、**低速设备**。
2. **轮流检查设备**，效率较低，但能同时服务很多设备。

------

### **数组选择通道**

- **核心点**：数组选择通道是专门服务**高速设备**（比如磁盘、磁带）的。
- 它一次只服务**一个设备**，但会传输**大批量的数据**（比如一个完整的文件或一块数据）。
- 任务完成后，再切换到下一个设备。

#### **为什么是选择的？**

因为通道会选择**一个设备**，专注地完成它的任务。

#### **特点**：

1. 一次只服务一个设备，但速度非常快。
2. 适合处理**大数据量**、**高速设备**。

------

### **数组多路通道**

- **核心点**：数组多路通道同时支持**低速设备**和**高速设备**。
- 对低速设备：它按顺序检查每个设备，像字节多路通道一样处理小数据。
- 对高速设备：它会像数组选择通道一样，传输大批量数据。

#### **为什么是多路的？**

因为它可以**同时管理多个设备**，灵活处理不同速度的设备。

#### **特点**：

1. 既能服务低速设备，又能服务高速设备。
2. 能同时处理多种任务，效率最高。

------

### **总结比较**

| **通道类型**     | **服务对象**      | **工作方式**                             | **适合场景**                |
| ---------------- | ----------------- | ---------------------------------------- | --------------------------- |
| **字节多路通道** | 低速设备          | 一个字节一个字节处理多个设备             | 键盘、打印机等低速设备      |
| **数组选择通道** | 高速设备          | 一次传输大批量数据，只服务一个设备       | 磁盘、磁带等高速设备        |
| **数组多路通道** | 高速+低速设备混合 | 同时支持逐字节和大批量传输，灵活处理设备 | 混合设备场景，如磁盘+打印机 |
