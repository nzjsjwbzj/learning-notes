# 死锁是

各进程相互等待对方手里的资源，导致各进程都阻塞，无法向前推进。因此如果有死锁现象，那至少有两个或两个以上的进程同时发生死锁。

# 死锁的产生有4个必要条件

：互斥条件，请求和保持条件，不剥夺条件，循环等待条件

互斥：一个资源每次只能被一个进程使用(资源独立)。

请求与保持：一个进程因请求资源而阻塞时，对已获得的资源保持不放(不释放锁)。

不剥夺：进程已获得的资源，在未使用完之前，不能强行剥夺(抢夺资源)。

循环等待：进程之间形成一种头尾相接的资源链(死循环)。

死锁必有环，有环不一定有死锁（资源节点里面可以有多个资源实例）

# 死锁有预防，避免，检测和解除4种处理策略

对于死锁的预防是破坏死锁产生的四个必要条件中的一个或几个，通常不会破坏互斥条件，因为可行性不高，并且很多时候无法破环
